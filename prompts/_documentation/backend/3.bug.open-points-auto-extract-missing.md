# Agent Orchestration — Open Points Not Auto-Extracted Bug

## Sorun

`open_points: []` her zaman boş dönüyor. BUILDER'a ve UI'a gap listesi ulaşmıyor.

**Sonuç:**
- User "açık noktalar" listesi göremiyor
- BUILDER gap'siz üretim yapıyor → `assumptions[]` boş çıkıyor
- User sonradan "ben Stripe demedim" gibi itirazlar yapıyor

## Kök Neden

`extract_open_points`, ANALYZE'ın bir **variant'ı**
([AnalyzeAgent.java:29](fp-api/ai/src/main/java/io/featureplanner/service/agent/handlers/AnalyzeAgent.java:29))
ama **otomatik tetiklenmiyor**. Sadece manuel
`action_hint_override: "ANALYZE:extract_open_points"` ile çağrılabilir.

Mevcut akış:

```
User mesaj → INTENT_ROUTER → ANALYZE:analyze_input
                              ↓
                              captured_topics, project_context güncellenir
                              ↓
                              open_points'i ÜRETMEZ (bu variant'ın işi değil)
                              ↓
                              extract_open_points HİÇ çağrılmaz
                              ↓
                              open_points = [] kalır
```

## Fix

`AnalyzeAgent.handle()` sonunda, uygun variant'larsa `extract_open_points`'i
sub-call yap:

```java
// AnalyzeAgent.java — state güncellemesi sonrası, response build etmeden önce:
if ("analyze_input".equals(action) || "analyze_document".equals(action)) {
    invokeExtractOpenPoints(state, invocation);
}
```

`invokeExtractOpenPoints` helper'ı:
1. Aynı `CompositionRequest` ile `extract_open_points` prompt'unu compose eder
2. LLM'e güncel state'i (topics + context dahil) yollar
3. Dönen `open_points[]` ve `project_notes[]` state'e merge edilir

**Alternatif:** Orchestrator'da ek dispatch — ama `AnalyzeAgent` içinde tutmak
daha kohezyonlu. ANALYZE kendi işini tamamlamış olur.

## Test

**T1 — analyze_input sonrası open_points dolu:**
- Given: `user_input = "Uber gibi, Stripe payment"`
- When: chat endpoint (INTENT_ROUTER → ANALYZE:analyze_input)
- Then: response'ta `projectState.openPoints.length > 0`
- Then: `agentChain` tek `ANALYZE` içerir (extract internal call)

**T2 — analyze_document sonrası:**
- Given: `file_text` dolu
- When: chat endpoint
- Then: `openPoints.length > 0`

**T3 — extract_open_points direkt çağrı:**
- Given: `actionHintOverride = "ANALYZE:extract_open_points"`
- When: chat endpoint
- Then: `openPoints` döner, diğer state değişmez (mevcut davranış korunur)

**T4 — diğer variant'lar etkilenmez:**
- Given: `actionHintOverride = "ANALYZE:add_input"`
- When: chat endpoint
- Then: `extract_open_points` **çağrılmaz** (sadece `analyze_input`/`analyze_document` için)

## Efor

~20 satır ekleme, `AnalyzeAgent.java` tek dosya.
