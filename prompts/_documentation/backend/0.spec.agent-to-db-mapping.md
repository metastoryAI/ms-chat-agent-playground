# Agent Output → DB Mapping

BUILDER / ANALYZE response'u DB'ye yazılırken uygulanan dönüşümler.

## Terminoloji Çakışması (Kritik)

Legacy DB isimlendirmesi prompt naming'i ile **örtüşmüyor**. Bu tablo ikisini ayırır:

| Kavram | LLM/Prompt alan adı | DB `feature` kolon adı |
|---|---|---|
| Kısa başlık (1-4 kelime) | `name` | `summary` *(String NOT NULL, 255 char)* |
| Uzun açıklama (1-2 cümle) | `summary` *(array)* | `description` *(TEXT)* |
| Açık teknik kararlar | `assumptions` *(array)* | `assumptions` *(yeni JSONB — eklenecek)* |
| Hiyerarşi ayırımı | yok (type=module/feature/subfeature) | `feature_type` *(GROUP / FEATURE / SUB_FEATURE)* |

**Unutma:** LLM tarafında `summary` = uzun açıklama. DB tarafında `summary` = title. Insert sırasında mapping şart.

---

## 1. Mapping Tablosu (element bazında)

### Module (featureType: `GROUP`)

| LLM alanı | Type (LLM) | DB kolon | Type (DB) | Dönüşüm |
|---|---|---|---|---|
| `name` | string | `summary` | String(255) | `name.substring(0, 255)` |
| `summary[].text` | string[] | `description` | TEXT | `summary[0]?.text ?? null` |
| `summary` array'i | `[{text, source, date}]` | (kaybolur) | — | history şu an persist edilmiyor |
| `assumptions` | string[] | `assumptions` | jsonb | as-is |
| `color` | string | `properties.color` | jsonb key | `{ ...properties, color }` |
| `id` *(synthetic `mod_*`)* | string | `feature_id` | String | client UUID üretir veya backend |

### Feature (featureType: `FEATURE`)

| LLM alanı | DB kolon | Dönüşüm |
|---|---|---|
| `name` | `summary` | substring(0, 255) |
| `summary[0].text` | `description` | flatten |
| `assumptions` | `assumptions` | as-is |
| `id` *(`feat_*`)* | `feature_id` | — |
| parent | `parent_id` | parent module'un UUID'si |

### Subfeature (featureType: `SUB_FEATURE`)

| LLM alanı | DB kolon | Not |
|---|---|---|
| `name` | `summary` | substring(0, 255) |
| `summary` *(plain string)* | `description` | flatten gerekmiyor, LLM zaten string yolluyor (bkz. `BUILDER_MODE_ADD_SUBFEATURES.md`) |
| `assumptions` | — | **Subfeature'da assumptions yok** — prompt üretmiyor |
| `id` *(`subfeat_*`)* | `feature_id` | — |
| parent | `parent_id` | parent feature UUID'si |

## 2. Flatten Helper

Module/Feature `summary` alanı LLM'de **array** (`[{text, source, date}]`), DB'de **string**. Tek noktada düzleştir:

```ts
// ms-ui/src/modules/boards/helpers/agent-tree-mapping.ts (yeni)

import type { AgentNs } from '@/namespaces/agent.namespace';

/** LLM summary array → DB description string. */
export function flattenSummary(summary: AgentNs.SummaryEntry[] | string | undefined | null): string | null {
  if (!summary) return null;
  if (typeof summary === 'string') return summary;           // subfeature şekli
  if (!Array.isArray(summary) || summary.length === 0) return null;
  return summary[0]?.text?.trim() || null;                   // ilk entry = en güncel
}

/** LLM name → DB summary (title, 255 char). */
export function toDbTitle(name: string | undefined | null): string {
  const n = (name ?? '').trim();
  return n.length > 255 ? n.substring(0, 255) : n;
}
```

**Neden ilk entry?** `summary[]` history array'i — **en güncel entry `[0]`'da** (prompt kuralı: yeni özet başa eklenir). Yayında değişirse `BUILDER_RULE_SUMMARY_STYLE.md`'ye bak.

**Kayıp:** `source` ve `date` DB'ye yazılmıyor. İlerleyen fazda `description_history` JSONB kolon açılırsa full array persist edilir.

## 3. Insert Path (initial, new elements)

BUILDER'ın ürettiği **yeni** elementler (`generate_modules`, `generate_modules_features`, `add_features`, `add_subfeatures`) plain `assumptions: [...]` array'i döner — delta değil.

### Payload (mapping uygulandıktan sonra)

```ts
// ms-ui/src/modules/boards/helpers/agent-tree-mapping.ts

export function mapModuleToInsertPayload(mod: AgentNs.Module) {
  return {
    featureType: 'GROUP',
    summary: toDbTitle(mod.name),
    description: flattenSummary(mod.summary),
    assumptions: mod.assumptions ?? [],
    properties: { color: mod.color },
    features: (mod.features ?? []).map(mapFeatureToInsertPayload),
  };
}

export function mapFeatureToInsertPayload(feat: AgentNs.Feature) {
  return {
    featureType: 'FEATURE',
    summary: toDbTitle(feat.name),
    description: flattenSummary(feat.summary),
    assumptions: feat.assumptions ?? [],
    properties: {},
    features: (feat.subfeatures ?? []).map(mapSubfeatureToInsertPayload),
  };
}

export function mapSubfeatureToInsertPayload(sf: AgentNs.Subfeature) {
  return {
    featureType: 'SUB_FEATURE',
    summary: toDbTitle(sf.name),
    description: flattenSummary(sf.summary),   // sf.summary plain string olabilir
    properties: {},
    // assumptions: gönderilmez — prompt üretmiyor
  };
}
```

### Endpoint

Mevcut: `POST /feature/add-groups-with-features`
- Backend DTO'su (`FeatureDto`) `assumptions` alanını **kabul etmiyor** şu an
- Backend'de DTO + entity + factory güncellenmeli (bkz. [4.task.java-builder-routing-and-modes.md](4.task.java-builder-routing-and-modes.md))

## 4. Diff Path (updated elements, `builder:diff` mode)

BUILDER `diff` mode iki tip çıktı döner:

### `new_modules[]` — yeni eklenenler
Initial insert path ile aynı (plain `assumptions: [...]`). Yukarıdaki mapping'i kullan.

### `updated_modules[]` — mevcut element üzerine delta

```json
{
  "id": "mod_payment",
  "summary": [{ "text": "...", "source": "doc_2" }],
  "added_assumptions": ["Stripe Connect vs direct charge not decided."],
  "resolved_assumptions": ["Payment provider not confirmed."],
  "features": [
    {
      "id": "feat_existing",
      "added_assumptions": [],
      "resolved_assumptions": []
    }
  ]
}
```

**Delta semantiği:**

| Kategori | Ne yapar |
|---|---|
| `added_assumptions[]` | Mevcut `assumptions`'a eklenecek |
| `resolved_assumptions[]` | Mevcut'tan **VERBATIM text match** ile kaldırılacak (çeviri/kısaltma yapılmaz) |
| Kept (ne added ne resolved) | Response'a **gelmez** — client zaten elinde tutar |

### Merge Helper

```ts
// ms-ui/src/modules/boards/helpers/agent-tree-mapping.ts

/**
 * Module/feature-scoped verbatim merge.
 * resolved_assumptions içindeki text existing listede birebir geçmeli.
 * Paraphrase / çeviri / kısaltma → match fail → assumption silinmez.
 */
export function mergeAssumptions(
  existing: string[],
  delta: { added_assumptions?: string[]; resolved_assumptions?: string[] }
): string[] {
  const resolved = new Set(delta.resolved_assumptions ?? []);
  const kept = existing.filter(a => !resolved.has(a));
  return [...kept, ...(delta.added_assumptions ?? [])];
}
```

**Scope:** Merge her zaman **module veya feature bazında** yapılır (global pool değil). Aynı assumption text'i farklı modüllerde duplicate olsa bile, `resolved_assumptions` sadece o modülün listesinde aranır.

**Verbatim kuralı nedeni:** LLM paraphrase yaparsa (`"Payment provider not confirmed."` → `"Payment provider undecided."`) client text-match fail eder → eski assumption silinmez → duplicate görünür. `BUILDER_MODE_DIFF.md` bu kuralı enforce ediyor ama cognitive safety için client tarafında da defensive olunmalı.

### Update Endpoint

Diff path için bulk update endpoint'i gerek:
```
POST /feature/update-features-bulk
Body: [{ featureId, summary, description, assumptions }]
```

Şu an `updateFeature` tek element alıyor — bulk version eklenmeli veya N defa çağrılmalı.

## 5. Edge Cases

### Summary array boş
```json
"summary": []
```
→ `flattenSummary` null döner → DB `description` null kalır (kolon nullable, sorun yok).

### Summary string gelir (subfeature)
```json
"summary": "Handles SMS delivery"
```
→ `flattenSummary` direkt string'i döndürür.

### Name 255 char'dan uzun
→ `toDbTitle` substring uygular. Kayıp kaçınılmaz (DB constraint). Prompt'ta "max 4 kelime" kuralı var ama defensive olunmalı.

### LLM `source` alanı yok
```json
"summary": [{ "text": "..." }]
```
→ Problem yok, description sadece text alır. Source şu an persist edilmiyor.

### Subfeature `assumptions` alanı doldu
Prompt üretmiyor ama LLM yanlışlıkla yollarsa → client mapping'de **drop edilir** (mapSubfeatureToInsertPayload'a koymadık).

### Synthetic ID (`mod_*`, `feat_*`, `subfeat_*`)
Insert öncesi client real UUID üretir (veya backend üretir ve response'ta döner). Preservation önemli değil — yeni element. Diff path'inde `id` = mevcut UUID, preserve edilir.

## 6. Backend Requirements (özet)

1. **DB migration** — `feature` tablosuna `assumptions jsonb` kolonu
2. **`Feature.java`** — `assumptions: List<String>` field (`@JdbcTypeCode(SqlTypes.JSON)`)
3. **`FeatureDto`** — `assumptions: List<String>`
4. **`FeatureFactory.create`** — assumptions propagate (create + update path)
5. **`/feature/add-groups-with-features`** — request path'inde assumptions accept
6. **`/feature/update-feature`** — update path'inde assumptions accept
7. **(opsiyonel)** `/feature/update-features-bulk` — diff path için

## 7. Testler (Client)

**T1 — flattenSummary array:**
- Input: `[{text: "A", source: "doc1"}, {text: "B"}]`
- Output: `"A"` (ilk entry)

**T2 — flattenSummary string:**
- Input: `"Plain text"`
- Output: `"Plain text"`

**T3 — flattenSummary empty:**
- Input: `[]`, `null`, `undefined`
- Output: `null`

**T4 — mergeAssumptions happy path:**
- existing: `["A", "B"]`, delta: `{added: ["C"], resolved: ["A"]}`
- Output: `["B", "C"]`

**T5 — mergeAssumptions verbatim miss:**
- existing: `["Payment provider not confirmed."]`
- delta: `{resolved: ["Payment provider undecided."]}`  ← paraphrase
- Output: `["Payment provider not confirmed."]` (match fail → silinmez)

**T6 — mergeAssumptions duplicate-safe:**
- existing: `["A"]`, delta: `{added: ["A"]}`
- Output: `["A", "A"]`  ← intentional, dedup module prompt'ta engellenir
- (Alternative: dedup'u client de yapabilir — karar ver)

**T7 — toDbTitle truncation:**
- Input: 300 karakterlik string
- Output: 255 karakter (substring)
