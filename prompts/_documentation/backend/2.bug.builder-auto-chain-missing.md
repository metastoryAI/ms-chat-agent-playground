# Agent Orchestration — BUILDER Auto-Chain Bug

## Sorun

User onayı olmadan BUILDER çalışıyor.

```
User: "Uber gibi bir uygulama"
agent_chain: [INTENT_ROUTER, ANALYZE, BUILDER]   ← User istemedi
```

## Kural

BUILDER **sadece 2 yolla** tetiklenmeli:

| Yol | Nasıl |
|---|---|
| Button tıklama | Frontend `action_hint_override: "BUILDER:generate_modules"` yollar |
| Chat'e yazma | User "generate" yazar → INTENT_ROUTER `intent: builder` döndürür |

Başka yol yok. Confidence ne olursa olsun.

## Kök Neden

`AgentOrchestrator.maybeChainFromAnalyze()` (satır 287-311), ANALYZE'ın emit ettiği
`[NA:GENERATE]` ve `[NA:BUILDER_TYPE|...]` tag'lerini **user onayı** gibi yorumluyor.

Bu tag'ler aslında **frontend için button hint'i** — "generate butonu göster" diyor.
Java bu tag'i yorumlamamalı, response'a koyup geçmeli. Button'a çevirme işini
frontend yapıyor (`next-actions.helper.ts`).

## Fix

`AgentOrchestrator.java` içinde:

1. **Sil:** `NA_BUILDER_TYPE` pattern (satır 44-45)
2. **Sil:** `NA_GENERATE` pattern (satır 48-49)
3. **Sil:** `maybeChainFromAnalyze()` metodu (satır 287-311)
4. **Sil:** `parseBuilderMode()` metodu (satır 319-328)
5. **Güncelle:** `chat()` içinde (satır 139-140):

```java
// ÖNCESİ:
AgentChatResponse chained = maybeChainFromAnalyze(route, primary, baseInv);
AgentChatResponse finalResp = chained != null ? chained : primary;

// SONRASI:
AgentChatResponse finalResp = primary;
```

6. **Temizle:** `fullChain` ve `fullUsedRows` içindeki `chained` referanslarını (satır 147, 153).

**Tag'ler response'ta kalır**, frontend okur ve button gösterir.

## Test

- Given: `user_input = "Uber gibi bir uygulama"`
