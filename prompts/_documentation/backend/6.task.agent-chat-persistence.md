# Agent Chat Persistence — Backend Task

## Sorun

Yeni `AgentOrchestrator` chat history'i DB'ye yazmıyor. Browser refresh edilince:
- Mesajlar kaybolur
- Pending `currentStructure` (commit edilmemiş tree) kaybolur
- `captured_topics`, `open_points`, `project_summary` kaybolur (ya da en son DB snapshot'ından restore olur — ama tur içi state gider)

**Referans:** [AgentOrchestrator.java](../../../fp-api/ai/src/main/java/io/featureplanner/service/agent/orchestrator/AgentOrchestrator.java) — repository injection yok, save call yok.

Eski `AiChatService` `chat_conversation_message` tablosuna yazıyor ama plain text shape (`request_message TEXT`, `response_message TEXT`). Yeni agent sistemine uymuyor — turn başına 2 LLM call (INTENT_ROUTER + hedef agent), her ikisi de yapısal JSON.

## Çözüm

Yeni tablo: `agent_chat_message`. Eski tabloyu kirletme (farklı amaç, farklı consumer'lar).

## Schema

### Migration

```sql
CREATE TABLE agent_chat_message (
    id BIGSERIAL PRIMARY KEY,
    workspace_id BIGINT NOT NULL,
    project_id BIGINT NOT NULL,
    user_id BIGINT NOT NULL,
    turn_index INTEGER NOT NULL,        -- conversation içindeki sıra

    -- Display fields (frontend refresh restore eder)
    user_input TEXT,                    -- "Uber gibi bir app"
    rendered_text TEXT,                 -- chat_response.text
    rendered_hint TEXT,                 -- chat_response.hint
    na_tag VARCHAR(120),                -- "[NA:GENERATE|CONFIDENCE:72]"
    structure_snapshot JSONB,           -- currentStructure (pending tree, varsa)

    -- Routing metadata
    intent VARCHAR(20),                 -- "builder"
    variant_key VARCHAR(40),            -- "add_features"
    agent_chain TEXT[],                 -- ["INTENT_ROUTER", "BUILDER"]
    action_hint_override VARCHAR(60),   -- direct route ise dolu

    -- Debug / replay payloads
    request_payload JSONB,              -- backend → LLM (her LLM call ayrı entry'de)
    response_payload JSONB,             -- LLM → backend

    status VARCHAR(20) DEFAULT 'ACTIVE',
    created_at TIMESTAMP NOT NULL DEFAULT NOW(),
    created_by BIGINT,

    CONSTRAINT fk_agent_chat_workspace FOREIGN KEY (workspace_id) REFERENCES workspace(id),
    CONSTRAINT fk_agent_chat_project FOREIGN KEY (project_id) REFERENCES project(id)
);

CREATE INDEX idx_agent_chat_project_turn ON agent_chat_message (project_id, turn_index DESC);
CREATE INDEX idx_agent_chat_user ON agent_chat_message (user_id);
```

**Not:** Bir turn'de birden fazla LLM call olabilir (INTENT_ROUTER + agent). İki yaklaşım:

- **A) Her LLM call ayrı satır.** `turn_index` aynı kalır, satırlar `agent_chain[]` içinde sıralanır.
- **B) Turn başına tek satır.** `request_payload` ve `response_payload` array olur (`[ {agent: 'INTENT_ROUTER', req, res}, {agent: 'BUILDER', req, res} ]`).

**B** önerilir — refresh için tek satır okumak yeter, `agent_chain` zaten ayrı kolonda. Debug için array yapı yeterli.

## Save Integration

`AgentOrchestrator.chat()` sonunda — final response build edilmeden önce:

```java
private final AgentChatMessageRepository chatMessageRepository;

public AgentChatResponse chat(AgentChatRequest request) {
    // ... mevcut akış ...

    AgentChatResponse response = ...;

    persistTurn(request, response, agentChainList, llmCalls);

    return response;
}

private void persistTurn(AgentChatRequest req, AgentChatResponse res,
                         List<String> chain, List<LlmCall> calls) {
    AgentChatMessage entry = AgentChatMessage.builder()
        .workspaceId(req.getWorkspaceId())
        .projectId(req.getProjectId())
        .turnIndex(nextTurnIndex(req.getProjectId()))
        .userInput(req.getUserInput())
        .renderedText(res.getChatResponse() != null ? res.getChatResponse().text() : null)
        .renderedHint(res.getChatResponse() != null ? res.getChatResponse().hint() : null)
        .naTag(res.getNextActions())
        .structureSnapshot(res.getCurrentStructure())
        .intent(res.getIntent())
        .variantKey(res.getVariantKey())
        .agentChain(chain)
        .actionHintOverride(req.getActionHintOverride())
        .requestPayload(calls.stream().map(LlmCall::request).toList())
        .responsePayload(calls.stream().map(LlmCall::response).toList())
        .status(BasicStatus.ACTIVE)
        .build();

    chatMessageRepository.save(entry);
}
```

`LlmCall` — internal record her LLM çağrısının req/res'ini biriktirir. `INTENT_ROUTER` + hedef agent için 2 entry doldurur. Direct route'ta (action_hint_override) tek entry.

**Async save (opsiyonel):** Latency endişesi varsa `@Async` ile paralel save. Hata durumunda log + chat akışını bloke etme.

## Retention Policy

İlk faz için:
- `status = ACTIVE` mesajlar tutulur
- Retention TTL yok (manual cleanup)
- GDPR delete user → cascade delete (foreign key veya scheduled job)

İleri faz:
- 90 gün+ eski mesajlar archive edilir veya silinir
- Storage threshold'u izlenir (request_payload + response_payload büyük olabilir)
- Cron job: `agent_chat_message` tablosunu monitor

## Testler (Given-When-Then)

**T1 — Happy path persist:**
- Given: chat request gelir
- When: `chat()` tamamlanır
- Then: `agent_chat_message` 1 yeni satır
- Then: `agent_chain`, `intent`, `variant_key`, `request_payload[]`, `response_payload[]` doğru dolu

**T2 — Direct route (action_hint_override):**
- Given: `actionHintOverride = "BUILDER:add_features"`
- When: chat
- Then: `agent_chain = ["BUILDER"]`, `request_payload` 1 entry
- Then: `action_hint_override` kolonu dolu

**T3 — Restore N mesaj:**
- Given: 60 mesaj var, limit 50
- When: GET `/agent/conversation`
- Then: en yeni 50 mesaj döner, turn order ascending

**T4 — Pending structure restore:**
- Given: BUILDER çıktı, kullanıcı commit etmedi, refresh
- When: restore endpoint
- Then: `latestStructure` son turn'ün `structure_snapshot`'ını döner

**T5 — Async save fail:**
- Given: DB save exception fırlatır
- When: chat() çalışıyor
- Then: response yine döner, error log'lanır, kullanıcı etkilenmez

**T6 — Workspace izolasyonu:**
- Given: user A'nın workspace 1'inde mesaj var
- When: user A workspace 2'den restore eder
- Then: workspace 1 mesajları **dönmez**

**T7 — Eski AiChat kirlenmedi:**
- Given: `chat_conversation_message` tablosu mevcut (eski sistem)
- When: agent-chat çalışır
- Then: yeni satır `agent_chat_message`'a yazılır, eski tablo dokunulmaz
