# LLM I/O Contract

Her agent için: (1) backend → LLM input JSON, (2) LLM → backend output JSON.

System prompt composition: `AgentPromptComposer` → `ai_agent_prompt` tablosu (kapsam dışı).

---

## Common Shapes

Tekrarlayan shape'ler — agent bölümlerinde `[...]` veya `{...}` referansları buradan okunur.

### `captured_topics[]`
```json
{ "title": "string", "summary": "string" }
```

### `open_points[]`
```json
{
  "id": "string",
  "text": "string",
  "element_name": "string",
  "element_id": "string | null"
}
```

### `inputs[]`
```json
{
  "id": "string",
  "type": "manual_input | free_input | document",
  "title": "string",
  "content": "string"
}
```

### `resolved_points[]`
```json
{ "question_id": "string", "answer": "string" }
```

### `project_context`
```json
{
  "summary": "string",
  "confidence": 0,
  "covered_concepts": ["string"],
  "gaps": ["string"],
  "platform_type": "string | null",
  "market_type": "string | null"
}
```

### `chat_response`
```json
{ "text": "string (markdown)", "hint": "string | null" }
```

### `summary_entry[]` (module/feature `summary` array içeriği)
```json
{ "text": "string", "source": "string | null", "date": "string | null" }
```
- Yeni üretimde `[{ text, source }]` (date client tarafından eklenir)
- Subfeature'da `summary` plain string (array değil)

### `sections[]` (BUILDER full output)
```json
{
  "id": "string",
  "name": "string",
  "modules": [
    {
      "id": "string",
      "name": "string",
      "summary": [{ "text": "string", "source": "string | null" }],
      "assumptions": ["string"],
      "color": "string",
      "features": [
        {
          "id": "string",
          "name": "string",
          "summary": [{ "text": "string", "source": "string | null" }],
          "assumptions": ["string"]
        }
      ]
    }
  ]
}
```
`features[]` sadece `generate_modules_features` ve `add_features` mode'larında dolu.

### `pages[]` (BUILDER:generate_pages output)
```json
{
  "id": "string",
  "type": "public | app",
  "name": "string",
  "summary": [{ "text": "string", "source": "string | null" }],
  "assumptions": ["string"]
}
```

### `targets[]` (BUILDER add_features / add_subfeatures input)

**add_features için (target = module):**
```json
{
  "id": "string",
  "name": "string",
  "summary": [{ "text": "string", "source": "string | null" }],
  "assumptions": ["string"],
  "color": "string",
  "existing_features": ["string"]
}
```

**add_subfeatures için (target = feature):**
```json
{
  "id": "string",
  "name": "string",
  "summary": [{ "text": "string", "source": "string | null" }],
  "assumptions": ["string"],
  "existing_subfeatures": ["string"],
  "parent_module": {
    "id": "string",
    "name": "string",
    "summary": [{ "text": "string", "source": "string | null" }],
    "assumptions": ["string"]
  }
}
```

## Direct Route — `action_hint_override`

Backend her zaman 2 LLM call yapar (INTENT_ROUTER → hedef agent), **istisna:**

```json
"action_hint_override": "AGENT:variant_key"
```

set ise INTENT_ROUTER atlanır, doğrudan `AGENT` + `variant_key` dispatch edilir. `agentChain = [AGENT]`.

Format örnekleri: `"BUILDER:add_features"`, `"ANALYZE:extract_open_points"`.

---

## INTENT_ROUTER

### Input
```json
{
  "user_input": "string",
  "project_summary": "string | null",
  "project_context": { },
  "existing_structure": { "sections": [] },
  "captured_topics": [],
  "open_points": [],
  "project_language": "string"
}
```

### Output
```json
{
  "intent": "answer | clarify | query | mutation | analyze | builder | interview",
  "variant_key": "string | null",
  "reason": "string"
}
```

`variant_key` sadece composite agent'lar (analyze / builder / interview) için. Standalone'larda null.

## ANSWER

```json

  "captured_topics": [],
  "open_points": [],
  "project_notes": [],
  "inputs": [],
  "workspace_id": "string",
  "project_id": "string"

```

### Output
```json

  "action": "answer",
  "chat_response": { "text": "string", "hint": "string | null" },
  "data": { "referenced_features": ["string"] },
  "next_actions": "string | null"

```

**Tools (read-only, 18):** `getApplicationContext`, `getProjectList`, `getProjectStats`, `getProjectPage`, `getModuleList`, `getFeatureList`, `getSubFeatureList`, `getContentOfFeature`, `getAcceptanceCriteriaOfFeature`, `getUserStoryOfFeature`, `getContentOfPage`, `getDescriptionOfScreen`, `getScreenList`, `getScreenImage`, `getFeaturesOfScreen`, `getGroupsOfScreen`, `searchFeatures`, `searchProjectPage`.

## BUILDER

Composite — `variant_key` (`mode`) ile variant seçilir.

### `generate_modules`

#### Input
```json
{
  "mode": "generate_modules",
  "user_input": "string",
  "project_summary": "string",
  "project_context": { },
  "captured_topics": [],
  "open_points": [],
  "resolved_points": [],
  "existing_structure": null,
  "is_regenerate": false
}
```

#### Output
```json
{
  "sections": [],
  "project_context": { },
  "chat_response": { "text": "string", "hint": "string | null" },
  "next_actions": "string | null"
}
```
`sections[].modules[]` dolu, `features[]` boş.

### `generate_modules_features`

#### Input
`generate_modules` ile aynı, `mode: "generate_modules_features"`.

#### Output
`generate_modules` ile aynı şema, `sections[].modules[].features[]` da dolu.

### `generate_pages`

#### Input
```json
{
  "mode": "generate_pages",
  "inputs": [],
  "project_context": { }
}
```

#### Output
```json
{
  "pages": [],
  "chat_response": { "text": "string", "hint": "string | null" }
}
```

### `add_features`

#### Input
```json
{
  "mode": "add_features",
  "targets": [
    {
      "id": "string",
      "name": "string",
      "summary": [{ "text": "string", "source": "string | null" }],
      "assumptions": ["string"],
      "color": "string",
      "existing_features": ["string"]
    }
  ],
  "inputs": [],
  "project_context": { },
  "resolved_points": []
}
```

#### Output
```json
{
  "modules": [
    {
      "id": "string",
      "features": [
        {
          "id": "feat_<synthetic>",
          "name": "string",
          "summary": [{ "text": "string", "source": "string | null" }],
          "assumptions": ["string"]
        }
      ]
    }
  ]
}
```
Yalnızca hedef modülün yeni feature'ları döner. Modül `id` preserve edilir, `name`/`summary`/`assumptions` değişmez.

### `add_subfeatures`

#### Input
```json
{
  "mode": "add_subfeatures",
  "targets": [
    {
      "id": "string",
      "name": "string",
      "summary": [{ "text": "string", "source": "string | null" }],
      "assumptions": ["string"],
      "existing_subfeatures": ["string"],
      "parent_module": {
        "id": "string",
        "name": "string",
        "summary": [{ "text": "string", "source": "string | null" }],
        "assumptions": ["string"]
      }
    }
  ],
  "inputs": [],
  "project_context": { },
  "resolved_points": []
}
```

#### Output
```json
{
  "features": [
    {
      "id": "string",
      "subfeatures": [
        {
          "id": "subfeat_<synthetic>",
          "name": "string",
          "summary": "string"
        }
      ]
    }
  ]
}
```
Subfeature `summary` plain string. Subfeature'da `assumptions` yok.

### `diff`

#### Input
```json
{
  "mode": "diff",
  "generation_type": "modules | modules_features",
  "existing_structure": {},
  "inputs": [],
  "project_summary": "string",
  "open_points": []
}
```

#### Output
```json
{
  "status": "diff_completed",
  "new_modules": [
    {
      "id": "mod_<synthetic>",
      "section_id": "string",
      "name": "string",
      "summary": [{ "text": "string", "source": "string | null" }],
      "assumptions": ["string"],
      "features": []
    }
  ],
  "updated_modules": [
    {
      "id": "string",
      "summary": [{ "text": "string", "source": "string | null" }],
      "added_assumptions": ["string"],
      "resolved_assumptions": ["string"],
      "features": [
        {
          "id": "string",
          "added_assumptions": ["string"],
          "resolved_assumptions": ["string"]
        }
      ]
    }
  ]
}
```

**Delta kuralı:** `resolved_assumptions[]` text'leri `existing_structure`'daki assumption'larla **VERBATIM** eşleşmeli (paraphrase / kısaltma / çeviri yasak — client text-match yapıyor).

### `resolve`

#### Input
```json
{
  "mode": "resolve",
  "existing_structure": {},
  "resolve_questions": [
    { "id": "string", "text": "string", "answer": "string" }
  ],
  "project_context": {}

```

#### Output
```json
{
  "status": "completed",
  "confidence": 0,
  "sections": []
}
```
