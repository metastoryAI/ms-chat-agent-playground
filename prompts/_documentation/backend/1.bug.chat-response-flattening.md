# Agent Orchestration — chat_response Flattening Bug

## Sorun

Backend LLM'den gelen `chat_response: { text, hint }` object'ini string'e
düzleştiriyor. **`hint` alanı frontend'e hiç ulaşmıyor.**

**Semptom:**
```json
// Frontend'e gelen response:
{
  "response": "Hello! Please describe...",   // ← sadece text
  "agent": "ANSWER",
  ...
  // hint yok
}
```

## Kural

Tüm agent'larda `chat_response` şekli **object** olmalı:

```json
{ "text": "string (markdown)", "hint": "string | null" }
```

Plain string yasak — frontend `hint` alanını button subtext'i veya ipucu
olarak kullanıyor, kaybolması UX bozuyor.

## Kök Neden

**1. DTO yanlış:** [AgentChatResponse.java:26-27](fp-api/ai/src/main/java/io/featureplanner/dto/agent/AgentChatResponse.java:26)

```java
@JsonProperty("response")
private String response;   // ← plain String, object değil
```

**2. Handler'lar düzleştiriyor:**
[AnswerAgent.java:129-144](fp-api/ai/src/main/java/io/featureplanner/service/agent/handlers/AnswerAgent.java:129)
(diğer handler'larda da aynı pattern):

```java
private static String extractChatText(Map<String, Object> map) {
    Object v = map.get("chat_response");
    if (v instanceof String s) return s;
    if (v instanceof Map<?, ?> nested) {
        return nested.get("text");   // ← hint atılıyor
    }
    ...
}
```

## Fix

### 1. Yeni DTO oluştur

`dto/agent/ChatResponseDto.java`:
```java
public record ChatResponseDto(
    @JsonProperty("text") String text,
    @JsonProperty("hint") String hint
) {}
```

### 2. `AgentChatResponse` güncelle

```java
// ÖNCESİ:
@JsonProperty("response")
private String response;

// SONRASI:
@JsonProperty("chat_response")
private ChatResponseDto chatResponse;
```

### 3. Handler'ları güncelle

Tüm 7 agent handler'da `extractChatText(...)` yerine `extractChatResponse(...)`:

```java
// Önce:
String chatText = extractChatText(llmResponse);
return AgentChatResponse.builder()
    .response(chatText)
    ...

// Sonra:
ChatResponseDto cr = extractChatResponse(llmResponse);
return AgentChatResponse.builder()
    .chatResponse(cr)
    ...
```

`extractChatResponse` helper'ı:
```java
private static ChatResponseDto extractChatResponse(Map<String, Object> map) {
    Object v = map.get("chat_response");
    if (v instanceof Map<?, ?> nested) {
        String text = (String) nested.get("text");
        String hint = (String) nested.get("hint");
        return new ChatResponseDto(text, hint);
    }
    if (v instanceof String s && !s.isBlank()) {
        // Legacy support — LLM plain string dönerse
        return new ChatResponseDto(s, null);
    }
    // Fallback: response / message keys
    String fallback = firstString(map, "response", "message");
    return fallback != null ? new ChatResponseDto(fallback, null) : null;
}
```

## Test

**T1 — hint dönüyor:**
- Given: LLM `{ "chat_response": { "text": "...", "hint": "tip" } }` döndü
- When: agent handle
- Then: response.chatResponse.text == "...", hint == "tip"

**T2 — hint null:**
- Given: LLM `{ "chat_response": { "text": "...", "hint": null } }` döndü
- Then: chatResponse.hint == null

**T3 — Legacy string (backward compat):**
- Given: LLM `{ "chat_response": "plain string" }` döndü (eski prompt)
- Then: chatResponse.text == "plain string", hint == null

**T4 — Fallback:**
- Given: LLM `{ "response": "old format" }` döndü
- Then: chatResponse.text == "old format", hint == null

## Efor

- 1 yeni record (ChatResponseDto)
- DTO güncelleme: 2 satır
- 7 handler güncelleme: her birinde ~3 satır
- Toplam ~30 satır değişiklik

## Frontend Etkisi

Frontend response parsing'i de güncellenmeli:

```ts
// Önce:
message.text = response.response;

// Sonra:
message.text = response.chat_response.text;
message.hint = response.chat_response.hint;
```

`chat-response` field ismi kırıcı değişiklik — frontend + backend aynı anda
deploy edilmeli veya geçiş süreci için backend her iki field'ı da
(`response` + `chat_response`) yollayabilir.
