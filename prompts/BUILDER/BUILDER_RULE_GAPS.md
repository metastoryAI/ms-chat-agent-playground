## GAPS
`gaps[]` topic_ids by platform type. Each topic represents a core decision area that affects which modules and features get generated.

- `app`: business_model · payments · user_roles · security · core_features · integrations · tech_stack · infrastructure
- `platform`: business_model · payments · user_roles · security · workflows · reporting · migration · integrations · infrastructure
- `website`: target_audience · business_model · core_features · payments · cms · seo · integrations
- `api_service`: endpoints · auth · data_model · consumers · integrations · rate_limiting · infrastructure
- `marketplace`: business_model · user_roles · trust_safety · payments · search · dispute_resolution · integrations
- `ecommerce`: catalog · checkout · payments · inventory · shipping · returns · integrations
- `saas_config`: modules_entities · field_structure · workflows · automation_rules · reporting · user_roles · migration · integrations · templates · permissions
- `automation`: triggers · actions · conditions · error_handling · scheduling · integrations · monitoring

Only include topic_ids from the matching platform_type list. Do not add topics from other lists or invent new ones.
