How it works
1
User input
Text message or PDF upload — pdf.js extracts text page-by-page into structured blocks
2
Build message
[DOCUMENT: file.pdf] <PARSED TEXT FOR PAGE: 1/N>... prepended to the state JSON
3
LLM call → Chat Agent
System prompt + message sent to selected provider. Returns JSON with action field
4
Frontend handles response
Updates in-memory state, renders message or structure card, may route to a second agent
5
Agent routing (optional)
Chat Agent can route to Structure Generator or Context Updater — new LLM call with full state
Active agents

Context Updater

Shared state sent to every LLM call
// buildPayload() — sent as JSON stringified user message { "documents": [{ id, name, original_summary, uploaded_at }], "manual_inputs": [{ id, topic, detail, added_at }], "project_summary": "combined text of all docs + inputs", "project_context": { summary, confidence, entities, covered, gaps }, "existing_structure": { pages: [], sections: [], modules: [], inserted_at }, "user_input": "what the user typed" }
PDF handling
PDFs are never sent as binary. pdf.js extracts plain text client-side and injects it into the message:
[DOCUMENT: myfile.pdf] <PARSED TEXT FOR PAGE: 1 / 11> Lorem ipsum... <PARSED TEXT FOR PAGE: 2 / 11> ...
Role
Entry point for every user message. Classifies the input and decides whether to analyze it, store it, answer directly, or route to a specialist agent.
Trigger: every user send Reads: full state Writes: documents, manual_inputs, project_summary
LLM output → actions
// analyze_document — file uploaded or PDF pasted { "action": "analyze_document", "document": { name, original_summary, topics }, "chat_response": "...", "next_actions": { ... } } → UI: push to state.documents, rebuild project_summary, show message // analyze_input — free text describes project { "action": "analyze_input", "chat_response": "...", "project_summary": "...", "next_actions": { ... } } // add_to_manual_input — new detail not yet in context { "action": "add_to_manual_input", "manual_input": { topic, detail }, "chat_response": "..." } → UI: push to state.manual_inputs, rebuild project_summary // modify_manual_input — correction to existing info { "action": "modify_manual_input", "manual_input_modification": { target_id, topic, new_detail } } // answer — direct reply, no state change { "action": "answer", "chat_response": "...", "next_actions": { label, buttons: [{ id, name, command, subtext }] } } // clarify — ask user something before acting { "action": "clarify", "chat_response": "...", "next_actions": { ... }, "pending_free_input": { ... } } // route_to_agent — hand off to specialist { "action": "route_to_agent", "agent": "structure_generator", "handoff": { generation_type, mode } } → UI: new LLM call to target agent with full state
Frontend state updates
state.documents.push(doc) state.manual_inputs.push(mi) state.project_summary rebuilt addAssistantMessage() updateStatePanel()
Role
Generates the app structure — pages and modules — rendered as an interactive structure card. Supports generate, refine, resolve assumptions, and diff modes.
Trigger: route_to_agent → structure_generator Reads: project_context, documents, manual_inputs, existing_structure Writes: state.existing_structure, state.project_context
Generation modes
modules
Pages + grouped modules with summaries
modules_features
Pages + modules + features per module
full
Pages + modules + features + subfeatures
refine
User gives instruction — only changes what was asked
resolve
Generates Q&A questions for open assumptions, user answers via QA widget
diff
Compares new context vs existing modules, returns only changed/new
LLM output → structure card
// status: completed — renders structure card { "status": "completed", "pages": [{ name, summary, assumptions[] }], "sections": [{ name, modules: [{ name, summary, assumptions[] }] }], "global_assumptions": [...], "confidence": 85, "assumption_label": "minor assumptions", "store_project_context": true, "project_context": { ... } } → UI: renderStructureCard() — collapsible rows, action buttons // status: diff_completed { "status": "diff_completed", "updated_modules": [...], "unchanged_count": 3 } → UI: show diff summary message // type: open_questions — for resolve assumptions { "type": "open_questions", "questions": [{ text, type, options: [{ id, label }] }] } → UI: showResolveWidget() → QA widget (qa.js)
Structure card actions
Resolve assumptions → QA widget for open questions Refine modules → user types instruction, re-generates Insert selected → saves to state.existing_structure
Frontend state updates
state.existing_structure = { pages, sections, modules } state.project_context (if store_project_context) renderStructureCard() — toggleable rows updateStatePanel()
